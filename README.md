# テスト採点システム

新入社員向け **バグ探しトレーニング教材** です。  
意図的にバグが仕込まれた採点アプリを動かしながら、仕様との差異を発見してください。

---

## 実行方法

`index.html` をブラウザで開くだけで動作します。サーバー不要。

```
kensyu-bug/
├── index.html   # HTML
├── style.css    # スタイルシート
├── script.js    # ロジック（JavaScript）
└── README.md    # 本ドキュメント
```

---

## 機能仕様

### 画面構成

| 要素 | 説明 |
|---|---|
| タイトル | 「テスト採点システム」 |
| 入力テーブル | 5科目の点数を入力するフォーム |
| 計算ボタン | バリデーションが全て通過した場合のみ活性になるボタン |
| 採点結果エリア | 計算ボタン押下後に表示されるセクション |

---

### 入力科目

以下の 5 科目それぞれに `<input>` フィールドが存在します。

| 科目 |
|---|
| 国語 |
| 数学 |
| 英語 |
| 理科 |
| 社会 |

---

### 入力バリデーション

各科目の入力フィールドに対して、以下の検証を行います。  
エラーが発生した場合は入力フィールドの右側にエラーメッセージを表示してください。

| 条件 | エラーメッセージ |
|---|---|
| 未入力（空文字）の場合 | `入力してください` |
| 数値以外が入力された場合 | `数値を入力してください` |
| 0 未満、または 100 を超える数値の場合 | `0〜100の整数を入力してください` |

> **ポイント**  
> - `0` は有効な値です。  
> - `100` は有効な値です（満点）。  
> - バリデーションはユーザーが **入力している最中** にリアルタイムで動作する必要があります。

---

### 計算ボタンの活性制御

- 全ての科目でバリデーションエラーがない場合 → ボタンを **活性（クリック可能）**
- 1 つでもバリデーションエラーがある場合 → ボタンを **非活性（クリック不可）**

---

### 採点結果の表示

計算ボタン押下時に、以下の 3 項目を表示します。

#### 合計点

5 科目の点数の合計値（単位: 点）

```
合計点 = 国語 + 数学 + 英語 + 理科 + 社会
```

#### 平均点

5 科目の点数の平均値（小数第 1 位まで表示、四捨五入）

```
平均点 = 合計点 ÷ 5
```

#### 判定

平均点をもとに以下の基準で判定します。

| 判定 | 条件 |
|---|---|
| **A** | 平均点が **80 以上** |
| **B** | 平均点が **60 以上 79 以下** |
| **C** | 平均点が **30 以上 59 以下** |
| **D** | 平均点が **29 以下** |

## バグ探しチャレンジ

このアプリには **合計 7 つのバグ** が意図的に仕込まれています。  
上記の仕様を正として、実装の誤りをすべて見つけてください。

### バグの分類

| カテゴリ | 件数 | ヒント |
|---|---|---|
| タイポによるエラー | 1 件 | ブラウザの開発者ツール（コンソール）が手がかりになります |
| 計算によるエラー | 2 件 | 実際に点数を入力して結果を確認してみましょう |
| イベント違いによるバグ | 1 件 | 入力中の動作と、フォーカスを外したあとの動作を比べてみましょう |
| 例外の考慮もれ | 1 件 | 通常の入力以外に、予想外の文字を試してみましょう |
| バリデーションの間違い | 1 件 | 境界値（0 や 100 など）を入力してみましょう |
| その他（初心者向け） | 1 件 | 高得点のケースで判定結果を仕様と照らし合わせてみましょう |

### 取り組み方

1. ブラウザで `index.html` を開く
2. 仕様どおりに動作しない箇所を探す
3. `script.js` のコードを読み、原因を特定する
4. 修正して正しく動作することを確認する

> 開発者ツールの **Console タブ** と **Elements タブ** を活用してください。
